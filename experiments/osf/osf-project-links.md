## Usage Rule

These links should be treated as external provenance anchors.

Before citing them in a manuscript or README as a specific preregistration, verify:

1. exact OSF project title;
2. public/private status;
3. registration status versus ordinary project status;
4. DOI or persistent identifier, if available;
5. latest file set;
6. whether the contents match the repo artifact being referenced.

## Claim Boundary

An OSF project link is a provenance record, not evidence by itself.

Evidence requires:

- preregistered protocol;
- fixed analysis plan;
- data or executable workflow;
- results;
- null/falsification interpretation;
- independent review or replication where possible.
