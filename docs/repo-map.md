# Repo Map

This document explains what each section of the repository is for and how future material should be added.

## docs/

Core narrative and structural documents.

- `repo-map.md` — this file
- `source-corpus.md` — index of source documents already folded into the repo
- `theory-overview.md` — concise research-facing summary of the framework
- `book-of-collapse-article-map.md` — maps long-form article/book material into modular research topics

## experiments/

Home for experimental design and prereg-ready material.

Planned contents:

- OSF preregistration drafts
- consciousness-coupled interferometry proposals
- delayed-choice reinterpretation notes
- quantum-computing observer-condition tests
- protocol versions, amendments, and materials lists

## simulations/

Home for computational and validation materials.

Planned contents:

- model equations translated into code
- notebooks
- parameter sweeps
- SPARC analysis notes
- validation charts
- black-hole / CMB / memory-field simulation notes
- imported screenshots and result summaries

## references/

Home for citations, validation notes, and comparative model framing.

Planned contents:

- bibliography
- observational datasets
- related interpretations of QM
- cosmology references
- notes on where SoCT agrees/disagrees with prior frameworks

## data/

Reserved for raw or semi-processed files.

Examples:

- CSV tables
- exported figures
- observational datasets
- simulation outputs
- processed parameter summaries

## notebooks/

Reserved for reproducible code analysis.

Examples:

- SPARC rotation curve fitting
- Hubble tension environment analysis
- CMB anomaly correlation tests
- symbolic or numeric memory-kernel experiments

## figures/

Reserved for rendered visuals and chart assets.

Examples:

- collapse sequence diagrams
- equation figures
- validation charts
- experiment schematics
- flowcharts and publication graphics

## papers/

Reserved for manuscript-grade writing.

Suggested future subfolders:

- `papers/foundations/`
- `papers/math/`
- `papers/cosmology/`
- `papers/experiments/`
- `papers/reviews/`

## import policy

When importing materials from threads or external files:

1. Preserve original wording where it matters.
2. Separate **published/public** material from **internal/pre-publication** material.
3. Label speculative material clearly.
4. Keep reproducible data and code separate from philosophical commentary.
5. Add provenance notes so each imported artifact can be traced back to its source.
